继承

继承格式：子类 extends 父类

继承内存原理

继承特点：单继承、不能继承构造器、不可访问父类私有成员、可共享父类静态成员

继承后成员变量、成员方法的访问特点（就近原则）

方法重写：建议加上重写校验注解@override

子类构造器的特点：默认访问父类无参构造器再执行

this和super

多态

定义：同类型的对象，执行同一个行为，会表现出不同的行为特征

常见形式： 父类类型 对象名称 = new 子类构造器

​ 接口 对象名称 = new 实现类构造器

多态中成员访问特点：

​ 方法调用：编译左边、运行右边

​ 变量调用：编译左边、运行左边

优势：可以实现解耦合，便于扩展和维护

问题：多态下不能使用子类的独有功能 可强制类型转换（有继承/实现关系的两个类型）来解决

内部类

访问作用域中的数据，包括私有数据。因为有一个外围类的引用OuterClass.this。可对一个包中的其他类隐藏。

静态内部类 声明： public static class Inner{}

​ 创建对象的格式：外部类名.内部类名 对象名 = new 外部类名.内部类构造器

简洁实现回调，如今lambda表达式在这方面做得更好。

匿名内部类：没有名字的局部内部类，为方便创建子类对象

接口

接口的基本使用：用来被类实现（implements），接口可以被类多实现

接口与接口可以多继承

JDK8允许接口中定义带有方法体的方法：

1.默认方法：default修饰 用实例对象调用

2.静态方法：static修饰，用接口名调用

static

  • static静态关键字

  • static应用知识

    工具类：静态方法，为完成共同功能。建议构造器私有化

​ 代码块：静态代码块

​ 格式：static{}

​ 特点：属于类，自动触发执行一次

​ 作用：程序启动时，进行静态资源的初始化操作

​ 单例设计模式

​ 1.饿汉单例设计：构造器私有，定义一个静态变量存储并创建一个对象

​ 2.懒汉单例设计——延迟加载对象 构造器私有，定义一个私有静态变量存储对象，提供一个公开方法返回单例对象
