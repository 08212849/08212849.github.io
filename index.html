git学习笔记

Created2022-08-24Updated2022-09-09

第一章 git认识

git是分布式版本控制系统。多人能在多地同时修改文件，因版本库在个人电脑，所以不需要联网就能修改并保留修改记录，可以随时查看过去的修改记录。多人间通过推送修改的方式到每个人的版本库，通常有一台充当“中央服务器”的电脑，用来方便“交换”大家的修改。

第二章 git基本命令

创建版本库

添加文件到版本库

注：所有的版本控制系统，智能跟踪文本文件的改动。windows自带的记事本编辑时会在每个文件开头添加0xefbbbf（十六进制）的字符，会遇到很多问题。

第一步，添加文件到仓库（可多次添加文件）

git add read.txt

第二步，提交文件到仓库

git commit -m "说明改动"

查看

当前仓库状态

git status

查看修改内容

git diff（difference）

使用git diff命令时涉及的概念

  • workspace:是本地项目的工作目录，属于本地代码更新但未执行git add命令时的状态，working tree的状态也随之更新。
  • index:索引文件，是连接working tree和commit的桥梁，当使用git add命令登记后，index file的内容会改变，这时index file 和working tree同步
  • local repository：是本地仓库，使用git commit命令时将index里的内容提交到本地仓库
  • remote repository：是远程仓库，当使用git push命令时会将本地仓库代码上传至远程仓库，完成整个代码的上传工作

git跟踪并管理的是修改，而非文件。修改需要被添加到缓存区，才能够被提交。

git diff readme.txt //查看workspace和index的差别

git diff --cached:查看index和local repository的差别

git diff HEAD:查看workspace和local repository（代码最新提交版本）的差别

显示从最近到最远的所有提交日志

git log //commit id(版本号)

回退

撤销上一次提交

git reset --hard HEAD^

HEAD表示当前版本，上一个版本为HEAD^,往上n个版本就是HEAD~n。

回到最新版本可以输入 git reset --hard 1040a 输入版本号前几位就可以

git reflog

记录每一次命令，找到commit id以便确定回到未来的版本

丢弃工作区的修改时，git checkout -- file

git checkout其实是用版本库里的版本替换工作区的版本，“一键还原”

丢弃暂存区的修改，第一步用命令 git reset HEAD <file> 回到丢失工作区的修改方式

删除

从版本库删除该文件：git rm + git commit。

注：先手动删除文件，使用git rm 和git add效果一样。

第三章 远程仓库

代码放置在本地电脑中，一旦出现故障或团队协作开发都很不方便，因此可以用github进行Git仓库免费托管，若想使用私有仓库需要付费或自己搭建Git服务器。

一般需要添加从本地库添加到远程库或从远程库克隆到本地库两个功能。

第四章 分支管理

在项目中，如果我们希望自己的工作不被别人修改，并且直到开发完毕后再一次性合并到原来的分支。既安全又不影响别人工作，因此需要创建自己的分支。

因为创建、合并和删除分支非常快，所以一般使用分支完成某个任务，合并后再删除。

关于分支的基本操作：

查看分支：git branch

创建分支：git branch <name>

切换分支：git checkout <name>

创建+切换分支：git checkout -b <name>或git switch -c <name>

合并某分支到当前分支：git merge <name>

删除分支：git branch -d <name>

发生冲突无法自动合并

需要手动解决冲突，将git合并失败的文件内容手动编辑为希望提交的内容后再提交。用git log --graph可以看到分支合并图。

合并分支时，加上--no-ff参数就可以用普通模式合并，合并后的历史有分支，能看出来曾经做过合并，而fast forward合并就看不出来曾经做过合并。
